Installation Usage MP3Tag Tags and Frames

Tags and Frames

Note: ID3v1 tags such as title, artist, album, year, comment, track, and genre will be overwritten by its equivalent ID3v2 frame if it is not blank.
Note: All string-typed frames are separated by '\\' and should not contain newline characters.
Text information frames
ID: "T000" - "TZZZ" excluding TXXX
Type: string
Links: ID3v2.3.0 section 4.2.1 and ID3v2.4.0 sections 4.2.1 - 4.2.5

If strict mode is enabled, all languages must be encoded with ISO 639-2. Some of these frames have its own standards but are only applicable if the strict mode is enabled.

  • Set frames such as "TPOS" and "TRCK" should follow this format: '1/2' or just '1' and can be separated '1/2\\1'.
  • Time frames at ID3v2.3.0 such as "TDAT", "TIME", "TORY", and "TYER" should have 4 numerical characters
  • Time frames at ID3v2.4.0 such as "TDEN", "TDOR", "TDRC", "TDRL", and "TDTG" should follow this format: yyyy-MM-ddTHH:mm:ss (year, "-", month, "-", day, "T", hour (out of 24), ":", minutes, ":", seconds), but the precision may be reduced by removing as many time indicators as wanted. Hence valid timestamps are yyyy, yyyy-MM, yyyy-MM-dd, yyyy-MM-ddTHH, yyyy-MM-ddTHH:mm and yyyy-MM-ddTHH:mm:ss.
  • "TKEY" frame is represented as a string with a maximum length of three characters. The ground keys are represented with "A","B","C","D","E", "F" and "G" and halfkeys represented with "b" and "#". Minor is represented as "m". Example "Cbm". Off key is represented with an "o" only.
  • "TSRC" frame should have 12 alphanumeric characters

User defined text information frame
ID: TXXX
Type: object[]
Links: ID3v2.3.0 section 4.2.2 and ID3v2.4.0 section 4.2.6

TXXX frames are parsed and must be saved as an array

          
            // TXXX example
            [{
              description: 'Text description', // description
              text: 'Any string here' // text information
            }]
URL frames
ID: "W000" - "WZZZ" excluding WXXX
Type: string
Links: ID3v2.3.0 section 4.3.1 and ID3v2.4.0 section 4.3.1

If strict mode is enabled, all URLs are checked if its a valid URL. The current RegExp used is /^(https?):\/\/[^\s/$.?#]+\.[^\s]*/

User defined URL link frame
ID: WXXX
Type: object[]
Links: ID3v2.3.0 section 4.3.2 and ID3v2.4.0 section 4.3.2

WXXX frames are parsed and must be saved as an array

          
            // WXXX example
            [{
              description: 'URL description', // description
              url: 'https://github.com' // URL
            }]
Comment and Unsynchronised lyrics frames
ID: COMM and USLT
Type: object[]
Links (COMM): ID3v2.3.0 section 4.11 and ID3v2.4.0 section 4.10
Links (USLT): ID3v2.3.0 section 4.9 and ID3v2.4.0 section 4.8

COMM and USLT frames are parsed and must be saved as an array. If the strict mode is enabled, there should only one frame with the same language and content descriptor. 

          
            // COMM and USLT example
            [{
              language: 'eng', // language
              descriptor: 'Content description', // content description
              text: 'Lyrics or comment goes here' // text
            }]
Attached picture frame
ID: APIC
Type: object[]
Links: ID3v2.3.0 section 4.15 and ID3v2.4.0 section 4.14

APIC frame is parsed and must be saved as an array. If the strict mode is enabled, there should only one frame with the same content descriptor, types should be in range of 0 - 21, format should match /(image\/[a-z0-9!#$&.+\-^_]+){0,129}/ regex, and description should not exceed 64 characters. 

          
            // APIC example
            [{
              format: 'image/jpeg', // Image MIME
              type: 3, // See the APIC image types at id3.org
              description: 'Content description', // content description
              data: [1, 2, 3, 4, 5, 6, 7, 8] // image bytes
            }]
General encapsulated object frame
ID: GEOB
Type: object[]
Links: ID3v2.3.0 section 4.16 and ID3v2.4.0 section 4.15

GEOB frame is parsed and must be saved as an array. If the strict mode is enabled, there should only one frame with the same content descriptor. 

          
            // GEOB example
            [{
              format: 'application/octet-stream', // Object MIME
              filename: 'filename.bin', // Filename
              description: 'Content description', // Description
              object: [1, 2, 3, 4, 5, 6, 7, 8, 9] // Object
            }]
Unique file identifier frame
ID: UFID
Type: object[]
Links: ID3v2.3.0 section 4.1 and ID3v2.4.0 section 4.1

UFID frame is parsed and must be saved as an array. If the strict mode is enabled, there should only one frame with the same Owner Identifier and it must be a non-empty string, and ID bytelength must not exceed 64 bytes. 

          
            // UFID example
            [{
              ownerId: 'http://www.id3.org/dummy/ufid.html', // non-empty owner ID
              id: [1, 2, 3, 4, 5, 6, 7] // binary identifier
            }]
Terms of use frame
ID: USER
Type: object
Links: ID3v2.3.0 section 4.23 and ID3v2.4.0 section 4.22

USER frame is parsed and must be saved as an object. If the strict mode is enabled, language must be encoded with ISO 639-2. 

          
            // USER example
            {
              language: 'eng', // language
              text: 'Text' // terms of use text
            }
Ownership frame
ID: OWNE
Type: object
Links: ID3v2.3.0 section 4.24 and ID3v2.4.0 section 4.23

OWNE frame is parsed and must be saved as an object. If the strict mode is enabled, date must match the format YYYYMMDD, currencyCode must be encoded with ISO 4217, and the currencyPrice must follow the format N.nn. 

          
            // OWNE example
            {
              currencyCode: 'USD', // US dollars
              currencyPrice: '2.00', // 2 dollars
              date: '20200424', // YYYYMMDD date format
              seller: 'Seller' // Seller's name
            }
Private frame
ID: PRIV
Type: object[]
Links: ID3v2.3.0 section 4.28 and ID3v2.4.0 section 4.27

PRIV frame is parsed and must be saved as an array. If the strict mode is enabled, there should only one frame with the same data 

          
            // PRIV example
            [{
              ownerId: 'owner string', // owners identifier
              data: [1, 2, 3, 4, 5, 6, 7, 8, 9] // data binary
            }]
Signature frame
ID: SIGN
Type: object[]
Links: ID3v2.4.0 section 4.28

SIGN frame is only available in ID3v2.4.0. SIGN frame is parsed and must be saved as an array. If the strict mode is enabled, all frames must be unique. 

          
            // SIGN example
            [{
              group: 1, // group ID
              signature: [1, 2, 3, 4, 5, 6, 7, 8, 9] // signature binary
            }]
Synchronised lyrics/text frame
ID: SYLT
Type: object[]
Links: ID3v2.3.0 section 4.8 and ID3v2.4.0 section 4.9

SYLT frame is parsed and must be saved as an array. If the strict mode is enabled, there should only one frame with the same language and content descriptor, language must be encoded with ISO 639-2, type should be in range of 0 - 6, format should be either 1 or 2, and lyrics must follow the format for each line: [mm:ss.xxx] String to display during the time 

          
            // SYLT example
            [{
              language: 'eng', // language
              format: 1, // Time stamp format
              type: 1, // Content type
              descriptor: 'Descriptions', // descriptions
              lyrics: 'Synced lyrics' // lyrics
            }]
Music CD identifier frame
ID: MCDI
Type: object
Links: ID3v2.3.0 section 4.5 and ID3v2.4.0 section 4.4

MCDI frame is parsed and must be saved as an object. 

          
            // MCID example
            {
              data: [1, 2, 3, 4, 5, 6, 7, 8] // binary data
            }
Synchronised tempo codes frame
ID: SYTC
Type: object
Links: ID3v2.3.0 section 4.8 and ID3v2.4.0 section 4.7

SYTC frame is parsed and must be saved as an object. If strict mode is enabled, format must be either 1 or 2. 

          
            // SYTC example
            {
              format: 1, // Timestamp format
              data: [1, 2, 3, 4, 5, 6, 7, 8] // binary data
            }
